# Known Bad Applications

## Apps that cause Quicksilver to crash

Certain applications have been seen to cause problems with Quicksilver. If you have any of the applications below installed on your system, try uninstalling them or updating as suggested.

  * **DefaultFolderX** — If you are using DefaultFolderX, please ensure that you update to the latest (4.4.7+) version to avoid crashes.
  * **Syncplicity** — On Lion 10.7.3, the `/Library/Contextual Menu Items/SyncpCMPlugin.plugin` causes Quicksilver to crash when trying to access any of the preferences.
  * **SCFinderPlugin** — To remove: make sure you have administrative privileges, then trash `/Library/Contextual Menu Items/SCFinderPlugin.plugin` and `/Library/Receipts/SCPlugin.pkg`.
  * **TotalFinder**

## Apps that interfere with Quicksilver

These applications are known to interfere with Quicksilver. Try disabling them to see if it fixes your problems.

  * Windowshade X
  * Spirited Away (though it should work fine as long as you add Quicksilver to Spirited Away's "Exclude List")
  * QuicKeys
  * Fruit Menu
  * USMLEWorld QBank (only if the QS clipboard plugin is installed)
